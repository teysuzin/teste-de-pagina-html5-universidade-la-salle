1. O que é Hypertext Markup Language (HTML)?

O HTML é a linguagem de marcação padrão para criar páginas na Web.
Ele funciona como o "esqueleto" de um site, definindo a estrutura básica do conteúdo, como títulos, parágrafos, links e imagens que os navegadores interpretam.

2. Como e por que surgiu o HTML?

O HTML surgiu no final da década de 80, criado por Tim Berners-Lee no CERN.
A ideia original era criar um sistema que permitisse aos cientistas e pesquisadores compartilhar documentos e pesquisas de forma rápida e organizada através de hiperlinks.

3. O que é eXtensible Hypertext Markup Language (XHTML)?

O XHTML é uma reformulação do HTML baseada na linguagem XML.
Ele foi desenvolvido para ser uma versão mais rígida e padronizada, exigindo que o código seja escrito com uma sintaxe perfeitamente limpa (como fechar todas as tags obrigatoriamente) para garantir melhor compatibilidade.

4. Qual a diferença de XHTML para HTML

O HTML é mais flexível e tolerante a erros. O navegador tenta “corrigir” automaticamente erros no código, não exige regras muito rígidas de sintaxe. Já o XHTML é uma versão do HTML baseada em XML, então segue regras bem mais rígidas, sendo obrigatório fechar todas as tags, tags essas que devem estar bem organizadas e aninhadas corretamente, tudo tem que estar em minúsculas, os atributos sempre precisam de aspas.

5. O que é eXtensible Markup Language (XML)

O XML serve pra organizar dados de um jeito que pessoas e máquinas consigam entender facilmente, aqui você mesmo define as “tags”. Isso que torna ele “extensível”, você cria a estrutura que quiser.

6. Qual a diferença de HTML para XML

A diferença no HTML e o XML está no objetivo e forma como são usados, o HTML é feito para exibir conteúdo (páginas web), já o XML é feito para armazenar e organizar dados. HTML usa tags pré-definidas, é focado em aparência e estrutura visual, e é mais flexível (tolera erros), já no XML, você cria suas próprias tags, é algo focado mais em dados e significado, e é bem rígido (tem que seguir regras certinhas)
